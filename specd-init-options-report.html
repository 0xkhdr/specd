specd init options report

Source: go run . help init, internal/cmd/init.go, internal/core/specfiles.go, docs.

Short read: specd init scaffolds .specd/ + AGENTS.md, then optionally configures coding-agent hosts and Brain/Pinky orchestration.

1) Core init variations

Option / variationWhat it leads tool to do
specd initDefault --agent auto. Detect hosts, scaffold project files, and in interactive TTY may configure 1 unambiguous host.
--agent autoDetect hosts. Interactive TTY: configure the one unambiguous host. Non-TTY or ambiguous set: scaffold only, show suggested actions.
--agent noneScaffold only. No host config mutation.
--agent <host>Target one supported host: codex, claude-code, cursor, antigravity, or vscode.
--agent allConfigure every detected supported host.
--yesAllows non-destructive project-scoped host registration without prompting. Required for non-interactive auto-detect to actually mutate host config.
--non-interactiveDisables prompts. With --agent auto and no --yes, init stays read-only on host config.
--scope projectDefault. Writes project-local host config only.
--scope globalGlobal host config path. Needs explicit consent; non-interactive global init requires --yes.
--dry-runNo writes. Prints planned actions like would write:, would update:, would preserve:.
--repairRestore missing managed assets only. Keeps user edits.
--refreshRefresh specd-managed assets and AGENTS.md markers. Keeps user-authored content outside managed sections.
--forceDestructively overwrite scaffold files and AGENTS.md.
--list-packsList embedded spec packs and exit.
--pack <name|url>Apply a pack. Remote URL requires pinned --sha256. Cannot mix with --dry-run, --repair, or --refresh.
--jsonEmit one versioned InitResult JSON document.
--verbosePrint detailed path results.

2) Orchestration options

FlagDefaultEffect
--orchestration [policy]planningEnables Brain/Pinky capability. Policies: manual, planning, session.
--orchestration-workers <n>4Sets orchestration.maxWorkers (clamped 1..64).
--orchestration-retries <n>2Sets orchestration.maxRetries (clamped 0..10).
--orchestration-timeout <minutes>120Sets orchestration.sessionTimeoutMinutes (clamped 1..1440).
--orchestration-cost-limit <usd>0Sets host-reported cost brake. Must be finite, non-negative.
--orchestration-mode <inline|delegate>delegateSets roles.subagentMode. delegate wants compatible host agents; inline is safer when no subagent host exists.
--orchestration-sandbox <none|bwrap|container>noneSets verify.sandbox. bwrap needs bubblewrap; container needs container support.

3) What orchestration init writes

  • .specd/config.json gets orchestration.enabled: true.
  • approvalPolicy becomes requested policy.
  • workerMode stays host.
  • transport defaults: file, poll 500ms, TTL 3600s, lease 120s, heartbeat 30s.
  • program.maxConcurrentSpecs defaults to 2.
  • If any host selected, result next step tells user to restart agent and run specd brain run <spec> --bootstrap --approval-policy ....

4) Advanced orchestration profile

Max autonomy profile: use session policy, high worker/retry caps, long timeout, delegate mode, and sandboxed verify.

Recommended command:

specd init --agent auto --yes \
  --orchestration session \
  --orchestration-workers 64 \
  --orchestration-retries 10 \
  --orchestration-timeout 1440 \
  --orchestration-cost-limit 0 \
  --orchestration-mode delegate \
  --orchestration-sandbox container

If host lacks container support, swap container for bwrap or none. If no compatible agent host exists, use --orchestration-mode inline or install one first.

5) Important behaviors / gotchas

  • --orchestration without value defaults to planning.
  • Invalid policy or bad numeric values return usage errors.
  • Values outside allowed ranges are clamped, with warnings.
  • --pack and --repair/--refresh/--dry-run are mutually exclusive.
  • --scope global needs explicit consent. In non-interactive mode, that means --yes.
  • --agent auto in non-interactive mode without --yes stays read-only on host config.
