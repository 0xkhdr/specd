## P0 gaps (blocking, per design "Release slices" P0)

1. **Wrong-root paths (R2.2).** `manifest.go:46-47` emits `specs/<slug>/...`; runtime lives at `.specd/specs/<slug>/`. Hosts load the wrong tree or nothing. Needs a declared repo base + canonicalization + traversal/symlink rejection.
2. **Missing required task knowledge (R2.1/R2.3).** No `design.md`, no declared source/test files, task item is `task_id` only (`:48`). Missing required files are silently skipped (`:91,94`) instead of failing with item identity.
3. **Untruthful budget (R3.1/R3.2).** Core items estimated from label bytes not content (`:52`); required overflow never fails because `lastDroppable` (`:137-155`) refuses to drop core yet the builder does not fail — false pass through `context-budget`.
4. **No typed manifest v2 (R1).** Flat `Item` with no trust/priority/reason/digest/load_mode lanes; `ManifestVersion="1"`; `ValidateManifest` does not fail closed on unknown kind/field/version (`:182-195`).
5. **No receipt/freshness (R5) and no context-pinned route/digests (R4.2/R4.3).** Palette/config digests exist in handshake (`handshake.go:41-46`) but are never pinned into the task manifest/evidence; no manifest digest exists, so staleness is undetectable.
6. **No production regression fixtures (R8.1).** Wrong-root, oversized-core, missing-design/source, stale-digest, and CLI/MCP route scenarios have no black-box coverage.
