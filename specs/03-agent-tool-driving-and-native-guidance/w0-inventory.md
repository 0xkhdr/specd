## P0 gaps (from Domain 03 P0 "make the existing contract truthful and executable")

1. **Context paths wrong (R2)** — `internal/context/manifest.go:46-47` emit top-level `specs/<slug>/...` instead of `.specd/specs/<slug>/...`; a driven agent reads the wrong file or nothing. No path-existence assertion in context tests.
2. **Manifest omits declared files + design (R2)** — `TaskRow.Files` and `design.md` never added to `BuildManifest`; host must rediscover task scope; `Item` struct (`manifest.go:16`) has no required/optional/reason/digest fields.
3. **Slugless managed commands (R3)** — `AGENTS.md:8` `specd status` and `:11` `specd verify` are not runnable against a fresh fixture; no placeholder marking convention; no extract-and-execute test over scaffolded guidance.
4. **Inert `SPECD_SPEC` pin (R4.2)** — `mcpconfig.go:42` writes it; no runtime reader; either wire a common resolver (explicit > pin > single, with source label) or stop emitting it. No pinned/explicit/missing/ambiguous black-box tests.
5. **No agent doctor (R5)** — no read-only `doctor --agent --json` producing stable findings for the four defects above; clean vs. defective fresh-fixture proof absent.

Secondary (needed for a truthful contract, P0/P1 boundary): handshake exposes no
guidance or context-schema digest (R7.1), so managed-template drift is invisible;
MCP authority refusal is an untyped string, not a typed handoff (R7.3).

## Boundary notes

- **Domain 01** owns phase legality, task rows, roles, approval semantics — R4/R6 consume, do not redefine them. (Aside: `manifest.go:157` `ModeForTask` maps scout/validator/scribe/craftsman but not `auditor`, which `AGENTS.md:20` advertises — a Domain 01/06 role-mapping gap, not owned here.)
- **Domain 02** owns manifest V2 selection/budget/schema and its digest; Domain 03 only exposes route + required-item semantics and consumes the context-schema digest for R1/R7.
- **Domain 04** owns evidence class/freshness; Domain 03 projects an immutable `evidence_ref` display only (R6 blockers, completion status). No completion authority here.
- **Domain 06** owns post-work role/tool/scope enforcement and actual authority; Domain 03 only *labels* `actor`/`authority_required` and renders the CLI handoff (R6.3/R7.3). `internal/mcp/server.go:87` forbidden-tool policy is the enforcement seam Domain 06/10 own.
- **Domain 10** owns MCP/adapter capability transport (R8.2) and remote envelope wire format; Domain 03 defines DispatchV1 field pinning (R8.3).
