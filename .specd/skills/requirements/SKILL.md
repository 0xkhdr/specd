# Requirements

## Instructions
Use stable requirement and criterion IDs. Write EARS behavior, owners, priority, risks, failures, edges, and non-goals.

## Examples
`R1.1: When input is invalid, system shall reject it before mutation.`

## Checks
Each behavior is testable, uniquely addressed, and free of implementation detail.
